Unifying graph databases
and causal models GO-Y · pronounced “go why”

Teaching graph databases to ask not just “what” and “how” — but “why”. GO-Y lays the theoretical foundations of causality-driven graph data management: a radical shift that makes causal reasoning a native, first-class capability of graph database systems.

The scientific gap

Two paradigms that have
never spoken to each other.

Causal scientists draw causal DAGs and reason over them with do-calculus and ad-hoc scripts. Graph databases query connected data at scale with declarative languages like Cypher and the ISO GQL standard. Yet the two communities work in isolation.

GO-Y closes this gap. Causal graphs are no longer hand-curated files living outside the database — they become first-class, queryable, versioned data. GO-Y turns causal reasoning from a bespoke scripting exercise into a rigorous, reproducible data-management discipline.

Research programme

Three pillars toward a causal graph database

From a formal model, to a query language, to the lifecycle of causal knowledge — GO-Y builds the full stack of causality-driven graph data management.

01 / Model

A formal causal property graph model

A rigorous data model that unifies the labelled property graph with the semantics of causal DAGs — making causal edges, interventions and assumptions first-class, typed, queryable elements of the graph rather than ad-hoc external files.

02 / Language

Causal operators in graph query languages

Extend declarative languages in the lineage of Cypher and ISO GQL with causal query operators — interventions (do()), confounder identification, d-separation and counterfactual “what-if” queries — bringing do-calculus into the database query layer.

03 / Lifecycle

Integration, consistency & versioning

Merge causal models from many experts and from observational data; detect and resolve contradictory assumptions; enforce acyclicity; and version causal graphs so analyses are reproducible, comparable and auditable as the science evolves.

What it looks like

Causal questions, asked declaratively.

Today, answering a counterfactual means writing bespoke scripts over a hand-drawn diagram. GO-Y’s vision: express interventions and “what-if” analysis directly in the query language, so they can be validated, optimised and reproduced like any other database query.

Illustrative syntax — part of the research agenda, not a released product.

causal-query · GQL+ 
MATCH CAUSAL PATH (s:Smoking)-[:CAUSES*]->(c:Cancer)
INTERVENE do(s.level = 'none')
ADJUST FOR confounders(s, c)
RETURN counterfactual(c.risk)
       AS averted_risk
